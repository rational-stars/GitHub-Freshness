# GitHub-Freshness 版本日志

## v2.0.0 - 2026年7月15日

### Chrome 扩展

- 新增 Manifest V3 Chrome 扩展版，支持同步保存设置，并可通过浏览器工具栏打开设置面板。
- Chrome 扩展改为通过官方渠道单独发布，公开仓库不再包含扩展源码和本地加载目录。
- 修复扩展重新加载后旧页面访问同步存储时可能产生的错误。

### 页面体验

- 重构仓库页面加载流程，保留 GitHub 原生加载状态，在文件列表首次显示前完成颜色、日期和排序处理，减少页面闪烁。
- 支持 GitHub 页面内导航、项目切换、搜索结果及动态加载内容。
- Code 按钮可使用当前主题的新鲜背景色和文字颜色；关闭相关选项时保留 GitHub 原生样式。
- 在仓库页 Code 按钮后和搜索页操作区增加 GitHub Freshness 图标，可快速打开设置面板。
- 搜索结果背景现在铺满整张卡片，描述、语言、Star 数量和更新时间会跟随文字颜色设置；仓库名称、Topic 和操作按钮保留 GitHub 原生样式。
- 修复 GitHub 页面结构更新后仓库列表、搜索结果和 Awesome 项目链接识别异常的问题。
- 修复文件及文件夹图标颜色被 GitHub 页面样式覆盖的问题。

### 设置与备份

- 新增 JSON 设置导入和导出；导出文件不包含 AWESOME token，导入时保留本地 token。
- 默认界面语言调整为中文，并保留自动、中文和英文选项。
- 切换语言后立即预览设置面板文案，保存后才写入本地配置。
- 浅色和深色主题设置分别暂存，切换主题时不会丢失尚未保存的修改。
- AWESOME 启用状态改为独立设置，不再跟随浅色或深色主题变化。
- 本地没有配置时使用新的默认配色和自动主题，已有配置保持原值。
- 保存或导入设置成功后自动刷新当前页面，使配置完整生效。

### 设置面板

- 使用新的 branch-clock 图标，统一脚本图标、设置标题和页面快捷入口。
- 设置面板采用新的“午夜薄荷”配色，并优化桌面与移动端布局。
- 将部分设置名称调整为更直观的“新鲜期限”“文字颜色”“文件图标”和“显示具体格式”。
- 为两个颜色选择器增加“新鲜颜色”和“过期颜色”悬浮提示。
- AWESOME 和 token 分别提供功能文档与令牌生成页面入口。
- 面板底部增加文档、GitHub 仓库、个人主页和 Telegram 交流群入口，并在最下方增加轻量 Star 支持提醒。

### 问题修复

- 修复 Pickr 颜色选择器样式加载及切换主题后颜色显示错误的问题。
- 修复 Chrome 将 AWESOME token 输入框误判为密码并弹出保存提示的问题。
- 优化 Awesome API 请求缓存、重复处理和限流提醒。
- 改进 GitHub 中文及英文界面下的更新时间解析。

## 🌟 **新增功能**：
版本更新日志： v1.0.1 - 2025年2月18日
- **文件排序**：
  - 可以根据需求对文件进行排序，看项目的时间更快了呢。
- **Awesome类型项目支持**：
  - 现在Awesome项目可以在列表中直接展示该项目的 star 数量和最近更新日期。
  - Awesome需要请求 github api。 GitHub 对未认证的用户（即没有使用令牌或授权的用户）对 API 请求数量设定了限制。如果你频繁地进行请求，会超出这个限制
  解决方案：
    1.	使用认证请求：
    使用 个人访问令牌（Personal Access Token） 可以显著提高请求限制，未认证的用户每小时最大请求次数为 60 次，而认证的用户每小时最大请求次数为 5000 次。
    2.  如何使用个人访问令牌：
    1.登录 GitHub，访问 GitHub 个人访问令牌生成页面。https://github.com/settings/tokens
    2.点击 Generate new token，选择适当的权限（你可以选择默认权限），然后生成一个新的令牌。
    3.复制刚刚生成的令牌到GitHub-Freshness设置页面的--AWESOME token--配置后面的输入框内并勾选复选框☑️就 OK 了。
